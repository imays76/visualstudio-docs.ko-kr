# <a name="c6263"></a>C6263
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

경고 C6263: 루프에서 _alloca를 사용 하 여 스택을 오버플로가 있습니다이  
  
 이 경고는 메모리를 할당 하는 루프에서 _alloca를 호출 스택 오버플로가 발생할 수를 나타냅니다. _alloca는 호출 함수가 종료 될 때 메모리가 해제만 없는 스택에서 메모리를 할당 합니다. 사용자 모드에도 스택 제한 되며 페이지 스택의 한 커밋 실패로 인해 스택 오버플로 예외입니다. `_resetstkoflw` 함수는 스택 오버플로 조건으로부터 복구하여 프로그램이 예외 오류와 함께 실패하는 대신 계속 실행되도록 합니다. 경우는 `_resetstkoflw` 를 함수 호출 하지 않으면 이전 예외 다음는 가드 페이지가 없습니다. 다음번에 스택 오버플로가 발생하는 경우에는 예외가 전혀 발생하지 않으며 프로세스는 경고 없이 종료됩니다.  
  
 호출 하지 않아야 `_alloca` 할당 크기 또는 반복 횟수를 알 수 없는 경우 스택 오버플로가 발생할 수 있기 때문에 루프 내에서. 이러한 경우 힙 메모리와 같은 다른 옵션을 고려 하거나 [c + + 표준 라이브러리](http://msdn.microsoft.com/library/a37d3ba3-58af-47c7-9ee2-441ccd7b77ee) 클래스입니다.  
  
## <a name="example"></a>예제  
 다음 코드에서는이 경고를 생성합니다.  
  
```  
#include <windows.h>  
#include <malloc.h>  
#include <excpt.h>  
#include <stdio.h>  
  
#define MAX_SIZE 50  
  
void f ( int size )  
{  
  char* cArray;  
  __try  
  {  
    for(int i = 0; i < MAX_SIZE; i++)  
    {  
      cArray = (char *)_alloca(size);  
  
     // process cArray...  
    }  
  }  
  __except(GetExceptionCode() == STATUS_STACK_OVERFLOW ?   
              EXCEPTION_EXECUTE_HANDLER : EXCEPTION_CONTINUE_SEARCH )  
  {  
     // code...  
     puts("Allocation Failed");  
    _resetstkoflw();  
  }  
}  
```  
  
 다음 코드에서는이 경고를 해결 하려면 malloc ()를 사용 합니다.  
  
```  
#include <windows.h>  
#define MAX_SIZE 50  
  
void f ( int size )  
{  
  char* cArray;  
  
  for(int i = 0; i < MAX_SIZE; i++)  
  {  
     cArray = (char *) malloc(size);  
     if (cArray != NULL)  
     {  
       // process cArray...  
       free(cArray);  
     }  
  }  
}  
```
