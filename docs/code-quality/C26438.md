# <a name="c26438-nogoto"></a>C26438 NO_GOTO
"'Goto' 하지 마세요."

**C + + Core Guidelines**: ES.76: goto 방지

'Goto'를 사용 하는 광범위 하 게 위험 하며 오류가 발생 하기 쉬운 방법으로 승인 됩니다. (예: 문법에서 생성 하는 파서)에서 생성 된 코드에만 적합 합니다. 최신 c + + 기능 및 Guidelines 지원 라이브러리에서 제공 하는 유틸리티를 사용 하 여 쉽게 'goto'를 완전히 피할 수 여야 합니다.

## <a name="remarks"></a>설명
- 이 규칙은 사용 되지 않습니다 되며 따라서 컴파일러에서 무시 하는 템플릿 코드를 제외 하 고 데드 코드에서 발생 하는 경우에 'goto'의 모든 항목 경고 합니다.
  - 경고 'goto'를 포함 하는 매크로 도달할 경우 노이즈가 발생할 수 있습니다. 현재 보고 메커니즘은 이러한 매크로 확장 되는 모든 인스턴스를 가리킵니다. 하지만 문제를 해결 수 있습니다. 또는 매크로 변경 하 고, 사용을 방지 하 고, 자세한 유지 관리 가능한 메커니즘을 활용 하 여 일반적으로 한 곳에서 수행 하는 수 있습니다.
    ## <a name="example"></a>예제
    매크로에 ' goto 정리'

```cpp
#define ENSURE(E, L) if (!(E)) goto L;

void poll(connection &c)
{
    ENSURE(c.open(), end);                  // C26438

    while (c.wait())
    {
        connection::header h{};
        connection::signature s{};
        ENSURE(c.read_header(h), end);      // C26438
        ENSURE(c.read_signature(s), end);   // C26438
        // ...
    }

end:
    c.close();
}
```

## <a name="example"></a>예제
'goto 정리' 매크로-gsl:: finally를 사용 하 여 대체

```cpp
void poll(connection &c)
{
    auto end = gsl::finally([&c] { c.close(); });

    if (!c.open())
        return;

    while (c.wait())
    {
        connection::header h{};
        connection::signature s{};
        if(!c.read_header(h))
            return;
       if(!c.read_signature(s))
            return;
        // ...
    }
}
```
